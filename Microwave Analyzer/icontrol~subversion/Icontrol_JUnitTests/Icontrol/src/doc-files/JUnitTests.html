JUnit Tests in iC

Starting with version around v1.3.469 JUnit tests were implemented in iC. To do so, the iC-Framework had to be decoupled from the View (which also comes in handy once the new GUI will be implemented). To do so, a new interface GUI_Interface was defined which declares all GUI related methods that are called by the iC-Framework. The most prominent of such member is the DisplayStatusMessage method.
In the course of decoupling it was discovered, that not all classes that accessed the View (mainly, if not exclusively, the DisplayStatusMessage method) had a M_GUI defined. Previously this was solved by declaring DisplayStatusMessage as static but this was no longer possible with the new Interface. To solve this, it was chosen to implement a getView in IcontrolApp which allows to get access to the GUI at runtime.
